# Paw Performance

Paw Performance is a macOS menu bar app that shows a running creature (Cat or Dog) whose animation speed/state reflects system load.

When you click the menu bar icon, a popover opens with friendly performance insights:

- CPU usage + top processes
- Memory usage + pressure
- Battery, disk, and network stats

## Features

- Menu bar-first UX (`NSStatusItem`)
- Anime-inspired kawaii visual style (playful, energetic, derpy, cozy)
- 4 stepped animation tiers based on CPU:
  - Idle (0-15%): Gentle walking/breathing
  - Trot (16-40%): Light jogging
  - Sprint (41-70%): Fast running
  - Panic (71-100%): Frantic/chaotic running
- Runner template system:
  - `cat` and `dog` packs included
  - New packs can be added by dropping a new folder under `Resources/Runners`
- SwiftUI popover with:
  - Status card (casual-friendly)
  - Detailed metrics panel (power-user friendly)
- Preferences persisted in `UserDefaults`

## Requirements

- macOS 14+
- Xcode 15+ (or Swift 5.10+ toolchain)

## Build and Run

From the project root:

```bash
swift build
swift run PawPerformance
```

## Runner Pack Format

Each runner is a folder under `PawPerformance/Resources/Runners/<runner-id>/`:

- `metadata.json`
- `idle.png`
- `trot.png`
- `sprint.png`
- `panic.png`

Example:

```json
{
  "id": "cat",
  "name": "Cat",
  "speedMultiplier": 1.0,
  "frameSize": { "width": 22, "height": 18 },
  "states": {
    "idle": { "frames": 4 },
    "trot": { "frames": 4 },
    "sprint": { "frames": 4 },
    "panic": { "frames": 4 }
  }
}
```

## Sprite Requirements

Each runner requires 4 PNG sprite sheets:

| State  | Frames | Dimensions | Description                        |
| ------ | ------ | ---------- | ---------------------------------- |
| idle   | 4      | 88 x 18    | Gentle walking/breathing animation |
| trot   | 4      | 88 x 18    | Light jogging motion               |
| sprint | 4      | 88 x 18    | Fast running                       |
| panic  | 5      | 110 x 18   | Frantic running                    |

### Sprite Sheet Format

- Each frame is **22 x 18 pixels**
- Frames are arranged horizontally in a single row
- Images must be **monochrome template** images (black silhouette with white eye highlights)
- All characters must face **LEFT**
- Keep all content within 22x18 frame boundaries (no cutoff/overlap)

### Visual Style

The app uses anime-inspired kawaii style sprites:

- Playful, energetic, derpy, cozy mood
- Black silhouettes (#2d2d2d) with white (#f0f0f0) eye highlights
- Keep proportions consistent across all frames in an animation

### Adding Custom Runners

1. Create a new folder under `PawPerformance/Resources/Runners/<your-runner>/`
2. Add the 4 sprite PNG files
3. Create a `metadata.json` with runner info:

```json
{
  "id": "your-runner",
  "name": "Your Runner",
  "speedMultiplier": 1.0,
  "frameSize": { "width": 22, "height": 18 },
  "states": {
    "idle": { "frames": 4 },
    "trot": { "frames": 4 },
    "sprint": { "frames": 4 },
    "panic": { "frames": 4 }
  }
}
```

## Troubleshooting

**Sprite doesn't appear or shows as placeholder:**

- Verify PNG dimensions are correct (88x18 or 110x18)
- Ensure images are template/monochrome (not colored)
- Check that frames face LEFT

**Animation looks wrong:**

- Ensure each frame stays within 22px width boundaries
- Verify frame count in metadata matches actual frames in PNG

**App won't build:**

- Ensure Xcode 15+ and macOS 14+
- Run `swift build` from project root
